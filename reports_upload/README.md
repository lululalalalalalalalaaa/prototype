# reports_upload — 보고서 업로드 폴더

이 폴더에 LCI DB 보고서 파일을 넣어 두면, 앱이 시작될 때(또는 사이드바의
**"📁 업로드 폴더 다시 스캔"** 버튼을 누를 때) 자동으로 학습(임베딩)합니다.

- 각 보고서는 **특정 LCI DB에 대한 정보**입니다.
- 이미 학습한 내용(이름 + 본문이 동일)은 건너뛰므로, 같은 파일이 있어도 중복 학습/중복 비용이 발생하지 않습니다.
- 폴더 안의 파일 원문은 앱 화면에 공개되지 않으며, 추천을 위한 학습용으로만 사용됩니다.
- 이 폴더의 보고서 파일은 비공개 자료이므로 git에 커밋되지 않습니다(`.gitignore` 처리).

## 지원 형식

### 0) `.hwp` — 한글(HWP 5.0) 파일 1개 = LCI DB 1개  ⭐ 기본
- **파일 이름**(확장자 제외)이 LCI DB 이름이 됩니다.
- 한글 문서의 **본문 텍스트**가 해당 LCI DB의 정보가 됩니다.
- 표·그림 등은 제외하고 본문 글자만 추출합니다.
- 참고: 신형 `.hwpx`(압축 XML) 포맷은 현재 미지원입니다. 한글에서 "다른 이름으로 저장 → HWP 형식(.hwp)"으로 저장해 주세요.

예) `철강 전기로 평균 데이터.hwp` → 파일명이 DB 이름, 문서 본문이 정보가 됩니다.

### 1) `.txt` / `.md` — 파일 1개 = LCI DB 1개
- **파일 이름**(확장자 제외)이 LCI DB 이름이 됩니다.
- **파일 내용** 전체가 해당 LCI DB의 정보가 됩니다.

예) `철강 전기로 평균 데이터.txt`
```
전기로 방식 조강 1톤 생산 기준 온실가스 배출 계수와
주요 투입 에너지·원료 인벤토리 정보를 담고 있다. ...
```

### 2) `.csv` — 행 1개 = LCI DB 1개
- **이름 열**과 **내용 열**이 필요합니다.
- 이름 열 머리글: `db_name`, `name`, `이름`, `LCI DB`, `데이터베이스` 중 하나
- 내용 열 머리글: `body`, `content`, `내용`, `정보`, `보고서` 중 하나
- 한글 깨짐 방지를 위해 UTF-8 인코딩 저장을 권장합니다(엑셀은 "CSV UTF-8"로 저장).

예) `lci_list.csv`
```csv
db_name,body
철강 전기로 평균 데이터,전기로 방식 조강 생산의 온실가스 배출 계수 ...
시멘트 클링커,클링커 소성 공정의 CO2 배출 인벤토리 ...
```

### 3) `.json` — 객체 목록 또는 단일 객체
- 각 객체에 이름 키(`db_name`/`이름` 등)와 내용 키(`body`/`내용` 등)가 있으면 됩니다.

예) `lci_list.json`
```json
[
  {"db_name": "철강 전기로 평균 데이터", "body": "전기로 방식 조강 생산의 온실가스 배출 계수 ..."},
  {"db_name": "시멘트 클링커", "body": "클링커 소성 공정의 CO2 배출 인벤토리 ..."}
]
```
